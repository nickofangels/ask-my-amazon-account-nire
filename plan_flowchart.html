Phase 1 — Discover

What's possible?

Research all SP-API report types for sales & optimization. Don't assume — build a probe first to confirm real access.

Phase 2 — Validate access

What do I actually have?

Fire lightweight create_report calls for all 13 candidates. Get a yes/no in seconds. Result: 12 accessible.

Phase 3 — Design

No duplication, best source wins

Scope to monthly / 2-year. Assign each metric to exactly one canonical table. Prevents conflicting data later.

Phase 4 — Infrastructure pivot

Portable from day one

SQLite is local-only. Supabase/PostgreSQL is accessible from any machine — caught this before writing a single fetcher.

Phase 5 — Execute

schema → backfill → validate

schema.py creates tables idempotently. backfill.py loops 24 months. Validate on 1 month before the full run.
